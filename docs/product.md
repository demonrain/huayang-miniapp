# 产品与业务规则

## 核心流程

1. 小程序启动后调用 `wx.login`，服务端识别微信 `openid`；首次登录赠送体验积分。
2. 用户按分类选择模板，进入创建页选择 1–6 张 JPG、PNG 或 WebP 图片。
3. 客户端显示每张单价和总积分，提交后服务端在同一事务内校验图片归属、扣除积分并创建任务。
4. Worker 按图片逐张调用生图供应商。全部成功后展示结果；任一异常则任务失败并自动退回本次全部积分。
5. 用户可查看历史任务、预览结果、批量保存到系统相册。
6. 积分不足时选择套餐并使用微信 JSAPI 支付；只有微信支付成功回调验签通过后才正式入账。

## 当前默认值

| 项目 | 默认值 |
| --- | --- |
| 新用户赠送 | 20 积分 |
| 单次上传 | 1–6 张 |
| 单图上限 | 10 MB |
| 模板单价 | 2–5 积分/张 |
| 登录有效期 | 30 天 |
| 任务失败 | 全额退回积分 |
| 分享到好友 | 默认 +2 积分/次（每日上限 3，后台可配） |
| 分享到朋友圈 | 默认 +1 积分/次（每日上限 1，后台可配） |
| 邀请新用户登录 | 默认邀请人 +5 积分（后台可配） |
| 邀请新用户完成首作 | 默认邀请人 +10 积分（后台可配） |

## 分享与邀请

- 分享领奖：用户分享自己的**已完成**作品时上报渠道（好友 / 朋友圈）；同一作品同一渠道每天最多发奖一次，且受每日上限约束。
- 邀请归因：被分享人打开分享页会记录 `inviteToken`；其**首次注册登录**时给邀请人登录奖；其**第一件成功作品**完成时给邀请人首作奖。
- 防刷：幂等 clientRequestId、每日上限、同作品同日去重；奖励开关与分值均可在管理后台配置。

## 关键不变量

- 用户不能提交其他用户上传的图片。
- 同一个 `clientRequestId` 只创建一个任务、只扣一次积分。
- 一个支付订单最多入账一次，重复通知直接返回成功。
- 生图失败退款以任务 ID 去重，同一任务最多退款一次。
- 服务端以图片文件头判断格式，不信任客户端 MIME 或文件扩展名。
- 客户端永远不保存微信 AppSecret、商户私钥、生图 API Key。
